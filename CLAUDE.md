# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

n8n community node (`@jaimeflneto/n8n-nodes-google-ads-conversion`) for uploading conversions to the Google Ads API. Published to npm; consumed by n8n via the `n8n` field in `package.json`, which points at compiled output in `dist/`.

## Commands

- `npm run build` — TypeScript compile + `gulp build:icons` (copies SVG assets to `dist/`).
- `npm run build:force` — Same, with `--skipLibCheck` (use when n8n-workflow type drift blocks a normal build).
- `npm run dev` / `build:watch` — `tsc --watch`.
- `npm run lint` / `lint:fix` — ESLint over `nodes` and `credentials` (`eslint-plugin-n8n-nodes-base`).
- `npm run format` — Prettier over `nodes`, `credentials`, `docs`.
- `npm test` — Runs `node tests/run-tests.js`, a custom validator (no Jest/Mocha). It loads `tests/test-data.json` and asserts node-structure invariants. There is no single-test runner; edit `tests/run-tests.js` or `tests/test-data.json` to scope a check.
- `npm run validate` — `test` + `build:force`. Run before publishing.
- `npm run package` — clean + force build + `npm pack`.
- `npm run prepublishOnly` — runs automatically on `npm publish`; does a clean build and validates.

Node engines: Node >=18, npm >=8. Package manager is npm (per `packageManager` field) — do not switch to pnpm/yarn.

## Architecture

The entire node is a single large file: `nodes/GoogleAdsConversion/GoogleAdsConversion.node.ts`. Understanding this file is the bulk of understanding the project.

- **Entry points** (declared in `package.json` `n8n` block):
  - `dist/credentials/GoogleAdsOAuth2.credentials.js` — OAuth2 credential type (`credentials/GoogleAdsOAuth2.credentials.ts`).
  - `dist/nodes/GoogleAdsConversion/GoogleAdsConversion.node.js` — the node itself.
- **Custom error classes** at the top of the node file (`GoogleAdsAuthenticationError`, `GoogleAdsValidationError`, `GoogleAdsApiError`, `GoogleAdsRateLimitError`) all extend `NodeOperationError`. New error paths should classify into one of these rather than throwing generic errors — retry/rate-limit logic depends on the class.
- **API target**: Google Ads REST API `v23` (`requestDefaults.baseURL`). When bumping versions, also check request body shapes for `:uploadClickConversions`.
- **MCC support**: The node detects manager vs. regular accounts and sets the `login-customer-id` header accordingly. Account selection uses `loadOptions`/`listSearch` methods (see `ILoadOptionsFunctions` import) — UI dropdowns are populated dynamically from the API.
- **Identification methods**: GCLID, Enhanced Conversions (with automatic SHA-256 hashing of PII via `crypto.createHash`), GBRAID, WBRAID. Hashing must happen before the payload leaves the node — never log unhashed user data.
- **Batch processing**: Up to 2000 conversions per request, with three modes (Partial Failure / Fail Fast / Continue on Error). Retry uses exponential backoff with jitter and respects `Retry-After` from rate-limit responses.
- **Validation mode** and **Debug mode** are node parameters; preserve them when adding new operations.

## Conventions specific to this repo

- All node parameters live inline in the `description.properties` array of the node class — there is no separate descriptions file. Keep related parameter groups together and use `displayOptions` for conditional visibility.
- Subtitle and parameter `displayName`s are user-facing in n8n — changing them is a UX change, not a refactor.
- TypeScript build uses `--noEmitOnError false`; `// @ts-ignore` comments exist intentionally for n8n version compatibility (e.g. `inputs`/`outputs` typing). Don't remove them without testing against multiple n8n versions.
- **`dist/` is gitignored** — it's a build artifact, not source. Never commit `dist/` files; they are generated by `npm run build` and shipped via the `files` field in `package.json` at publish time (`prepublishOnly` enforces a clean build).
